**Generated:** 2026-03-31
**Status:** READ-ONLY AUDIT

## Executive Summary

The card data system has **significant data duplication** and an **unused auto-generated file**. Key findings:

1. **card-v3-data-generated.ts is ORPHANED** — auto-generated but never imported or used
2. **9 cards missing from generated.ts** but present in both index and manual data.ts
3. **Data conflicts exist** between data.ts and generated.ts for overlapping cards (different baseRate values)
4. **Redundant baseRate storage** — stored in index, data.ts, and (unused) generated.ts
5. **2 portal cards missing from index** — icici-ishop, sbi-rewardz appear in portal eligibility but not in card catalog

## Conclusion

The card data system works functionally but has **significant redundancy and technical debt**:

1. **Unused 21K-line file** that creates confusion
2. **baseRate duplicated 3 ways** across 2 different counts
3. **1 unreachable card** in data.ts (184 vs 183 index)
4. **9 card gaps** between data files
5. **Data conflicts** on overlapping cards (different baseRate values)

**Recommended Action Plan:**
1. Delete card-v3-data-generated.ts immediately
2. Reconcile 184 vs 183 card count
3. Make card-v3-data.ts the single source for enrichment (not index)
4. Add schema validation to detect future mismatches
5. Document which file is authoritative for each data field
